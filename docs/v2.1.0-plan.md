# v2.1.0 實作計畫 - 完善示範資料

## 目標

讓使用者能**順暢走完完整的 ERP 流程**：
```
採購單 → 收貨 → AP發票 → 付款
銷售單 → 出貨 → AR發票 → 收款
```

## 實作順序與依賴關係

```
┌─────────────────────────────────────────────────────────────┐
│                      Phase 1: 基礎設定                        │
├─────────────────────────────────────────────────────────────┤
│  1. 付款條件 ──────┬──→ 2. 銀行帳戶                           │
│                    │                                         │
│                    └──→ 3. BP地址 + 4. 統編                   │
│                              │                               │
│                              ▼                               │
├─────────────────────────────────────────────────────────────┤
│                      Phase 2: 資料初始化                      │
├─────────────────────────────────────────────────────────────┤
│                    5. 初始庫存                                │
│                         │                                    │
│                         ▼                                    │
│                    6. 示範交易                                │
└─────────────────────────────────────────────────────────────┘
```

**依賴關係說明：**
- 付款條件：無依賴，可先做
- 銀行帳戶：需要幣別（系統已有 TWD）
- BP 地址/統編：需要現有 BP（已有）
- 初始庫存：需要商品、倉庫（已有）
- 示範交易：需要以上全部完成
